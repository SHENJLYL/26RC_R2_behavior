# R2 Action Preparation

本目录用于归档 R2 行为树后续接入任务级 action、service、topic 时的接口准备内容。

这些文档只做接口与任务边界设计，不新增核心代码。后续实现时应优先复用已有 R2 导航、视觉、机械臂、吸盘、底盘和安全节点；只有确认没有可复用接口时，才新增最小包装节点。

## 来源

本文档综合以下信息整理：

- 当前 `26RC_R2_behavior` 行为树 XML 和 md 文档。
- 当前已接入的 `NavigateToNamedPose`、`PubNav2Goal`、`PublishTwist`、`IsManualStart` 节点。
- 赛事规则 PDF 中关于 `武馆(MC)`、`梅林(MF)`、`树林`、`对抗区`、`端头`、`KFS`、`九宫格` 的约束。
- 当前 R2 车辆结构：四轮麦轮、四轮丝杠抬升、轮端红外高度检测、前方激光雷达、前方深度相机、机械臂夹爪、机械臂吸盘、机械臂深度相机。

## 设计原则

1. 行为树只做任务编排，不直接实现视觉识别、机械臂轨迹、树林方块规划和底盘控制细节。
2. 复杂能力优先做成外部任务级 action/service，再由行为树调用。
3. 状态类信息优先做成 topic，行为树通过 condition/action wrapper 读取汇总状态。
4. 规则检查应独立成 `rule_guard`，不要散落在每个动作节点里。
5. R2 只能携带一个由吸盘吸取的 KFS，`payload_state` 必须作为全局约束。
6. R1 二维码只作为可见时的状态来源，必须保存最近可信状态和时间戳，不能假设全程可见。
7. 所有动作都应返回可解释的失败原因，便于 rosbag 回放和赛后调研。

## 当前优先级

建议按以下顺序接入：

1. 健康检查、急停、载荷状态、规则检查。
2. 端头识别夹取与兵器组装。
3. R1 状态读取和离开武馆/进入梅林判断。
4. 树林 KFS 地图、方块规划、轮端抬升/找平、相邻 KFS 抓取。
5. 对抗区九宫格识别、R1 二维码辅助对齐、中层放置。
6. 重试恢复和异常流程。

## 术语

本文档采用规则 PDF 中的正式术语：

| 术语 | 说明 |
|---|---|
| `端头` | 武馆端头架上的兵器部件，代码旧标识可暂保留 `spearhead` |
| `梅林(MF)` | 二区，包含树林、R1 通道、R2 入口区和 R2 出口区 |
| `树林` | 梅林内 12 个编号方块区域，是 R2 收集 R2 KFS 的主要区域 |
| `KFS` | 武术秘籍，包含 R1 KFS、R2 KFS、假KFS |
| `九宫格` | 对抗区 3x3 架子，R2 当前目标是中层放置 |
